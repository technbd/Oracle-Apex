### Configuring Oracle APEX Images:


_Verify APEX Images Directory:_
```
ls /home/oracle/oracle-apex/apex/images
```



```
mkdir -p /home/oracle/ords/static/WEB-INF

cd /home/oracle/ords/static/WEB-INF
```


_Create file `web.xml`:_
```
vim web.xml

<?xml version="1.0" encoding="UTF-8"?>

<!DOCTYPE web-app PUBLIC
"-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN"
"http://java.sun.com/j2ee/dtds/web-app_2_3.dtd">

<web-app>
 <!-- This Web-App leverages the alternate doc-root functionality in WebLogic to serve static content
      For WebLogic refer to the weblogic.xml file in this folder
  -->
</web-app>
```


_Create file `weblogic.xml`:_
```
vim weblogic.xml

<weblogic-web-app xmlns="http://www.bea.com/ns/weblogic/weblogic-web-app">
 <!-- This element specifies the context path the static resources are served from -->

 <context-root>/i</context-root>

 <virtual-directory-mapping>
  <!-- This element specifies the location on disk where the static resources are located -->
  <local-path>/home/oracle/oracle-apex/apex/images</local-path>
  <url-pattern>/*</url-pattern>
 </virtual-directory-mapping>

</weblogic-web-app>
```


_Create a Web Application with the `web.xml` and `weblogic.xml` files:_
```
cd ..

jar cMf i.war WEB-INF/web.xml WEB-INF/weblogic.xml
```

### Configuring WebLogic to Handle HTTP Basic Challenges Correctly: (Optional)

```
cd /u01/app/oracle/product/14.1.1/oracle_common/common/bin

./wlst.sh

connect('wladmin','admin123','t3://10.1.1.11:7001')
cd('SecurityConfiguration')
cd('adminDomain') 
ls()
```


_If the domain settings displayed, contains the following entry:_
```
-r--   EnforceValidBasicAuthCredentials             true
```


_Then you must set this entry to `false`:_
```
connect('wladmin','admin123','t3://10.1.1.11:7001')
edit()
startEdit()
cd('SecurityConfiguration')
cd('adminDomain')  
set('EnforceValidBasicAuthCredentials','false')
save()
activate()
disconnect()
exit()
```



> [!NOTE]
> WebLogic Server must be restarted for the new settings to take effect.

### Deploy `ords.war` and `i.war` via WebLogic Admin Console: 

1. Login as weblogic admin: 
2. Left panel → click `Lock & Edit`
3. Left panel → Select `Deployments` → `Install`
4. Upload Path: `/home/oracle/ords` and `/home/oracle/ords/static`
    - Next
5. Select war file → `ords.war` and `i.war`
    - Next
6. Select → [✔] `Install this deployment as an application`
    - Scope: Global
    - Next 
7. Optional Settings: 
    - General: 
        - Name: `ords`
    - Security:
        - [✔] DD Only: Use only roles and policies that are defined in the deployment descriptors.
    - Source Accessibility:
        - [✔] Use the defaults defined by the deployment's targets
        - Location: `/home/oracle/ords/ords.war` and `/home/oracle/ords/static/i.war`
    - Plan Source Accessibility:
        - [✔] Use the same accessibility as the application
    - Next

8. Additional Configuration:
    - [✔] Yes, take me to the deployment's configuration screen.
    - Summary: 
    - Target Summary:
        - Components: `ords` | Targets: `AdminServer`
    - Click `Finish`
9. Settings for ords:
    - Overview: 

10. Click `Activate Changes`

11. Summary of Deployments - click `Deployments` - `Control`:
    - Select: `ords`
    - Click `Start` and `Servicing all requestes`
    - Click `Yes`

### Deploy `weblogic.Deployer` (Command Line): 


```
. ${WL_HOME}/server/bin/setWLSEnv.sh
```


```
java weblogic.Deployer -version

weblogic.Deployer version WebLogic Server 14.1.1.0.0
```

_Deploy ORDS WAR:_
```
java weblogic.Deployer \
  -adminurl t3://10.1.1.11:7001 \
  -username wladmin \
  -password admin123 \
  -deploy \
  -name ords \
  -source /home/oracle/ords/ords.war \
  -targets AdminServer
```


_Verify List of apps:_
```
java weblogic.Deployer \
  -adminurl t3://10.1.1.11:7001 \
  -username wladmin \
  -password admin123 \
  -listapps
```


_Undeploy ORDS:_
```
java weblogic.Deployer \
  -adminurl t3://10.1.1.11:7001 \
  -username wladmin \
  -password admin123 \
  -undeploy 
  -name ords
```
